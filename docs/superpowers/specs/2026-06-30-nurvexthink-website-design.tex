\section{Overview}
NurvexThink is a software company that builds and publishes software products, takes custom
software orders on demand, and writes blog posts about its work. This spec covers the
\textbf{main website} --- the public hub that showcases products (with links to their live
apps), hosts the blog, captures custom-software leads, and provides an admin dashboard for the
team to manage content.

\textbf{Out of scope (separate future specs):} the individual software products themselves.
Each product is its own repository and deployment, linked to from this site.

\subsection{Goals}
\begin{itemize}
  \item Premium, dark/metallic brand presence with signature 3D visuals.
  \item Showcase published software with links to live apps.
  \item Blog (created/edited via an admin dashboard --- no coding required to publish).
  \item Capture custom-software leads via a simple form.
  \item \$0 to launch (free tiers), with a clear paid-upgrade path.
\end{itemize}

\subsection{Non-goals (deferred)}
\begin{itemize}
  \item Client accounts / order-tracking dashboard (lead form only at launch).
  \item Online payments (Stripe) --- later phase.
  \item Multi-language / i18n.
\end{itemize}

\section{Architecture}
\textbf{Approach A (chosen): Next.js + Supabase on Vercel. No separate backend server.}

\begin{lstlisting}
[ Browser ]
    |  HTTPS
[ Next.js 15 App Router on Vercel ]
    |  - React Server Components (data fetching, SEO)
    |  - Client Components (3D, interactivity) at the leaves
    |  - Route Handlers / Server Actions (form mutations)
    |
[ Supabase Cloud ]
    - Postgres (data)         - Auth (admin login)
    - Storage (images)        - Edge Functions (custom logic, if needed)
    - Row Level Security (RLS) on every table
\end{lstlisting}

\section{Data model (Supabase)}
\subsection{Tables}
\begin{itemize}
  \item \texttt{profiles} --- \texttt{id (= auth.users.id)}, \texttt{email}, \texttt{full\_name},
        \texttt{role} (owner / admin), \texttt{created\_at}
  \item \texttt{products} --- \texttt{id}, \texttt{slug} (unique), \texttt{name}, \texttt{summary},
        \texttt{description}, \texttt{cover\_image}, \texttt{live\_url}, \texttt{repo\_url},
        \texttt{tags}, \texttt{featured}, \texttt{published}, timestamps
  \item \texttt{blog\_posts} --- \texttt{id}, \texttt{slug} (unique), \texttt{title}, \texttt{excerpt},
        \texttt{content}, \texttt{cover\_image}, \texttt{author\_id}, \texttt{status}
        (draft / published), \texttt{published\_at}, timestamps
  \item \texttt{orders} --- \texttt{id}, \texttt{name}, \texttt{email}, \texttt{company},
        \texttt{project\_type}, \texttt{budget}, \texttt{details}, \texttt{status}
        (new / contacted / closed), \texttt{created\_at}
\end{itemize}

\subsection{Storage buckets}
\begin{itemize}
  \item \texttt{product-images} (public read, admin write)
  \item \texttt{blog-images} (public read, admin write)
\end{itemize}

\subsection{Row Level Security (enabled on EVERY table, default-deny)}
\begin{itemize}
  \item \texttt{products}/\texttt{blog\_posts}: public SELECT where published; admins full access.
  \item \texttt{orders}: public INSERT only (with validation); admins SELECT/UPDATE; no public read.
  \item \texttt{profiles}: a user reads/updates own row; admins read all.
  \item Index every column referenced in an RLS policy (e.g.\ \texttt{published}, \texttt{status}, \texttt{author\_id}).
\end{itemize}

% ===================================================================
\section{Security}
\begin{itemize}
  \item RLS on every public-schema table, no exceptions (Supabase is default-deny).
  \item \texttt{service\_role} key is server-side only --- never in client code or the repo;
        stored as a Vercel / GitHub Actions secret.
  \item \texttt{anon} key is public-safe because RLS protects the data.
  \item Lead form: validate input server-side; add basic anti-spam (honeypot / rate limit).
  \item Admin routes guarded by Supabase Auth + middleware; \texttt{/admin} checks \texttt{profiles.role}.
\end{itemize}

\section{Deployment \& GitHub}
\begin{itemize}
  \item \textbf{Repo:} \texttt{nurvexthink/nurvexthink-web} (public).
  \item \textbf{Vercel project:} named independently (e.g.\ \texttt{nurvexthink}); custom domain later.
  \item \textbf{Deploy:} GitHub Actions builds and deploys with a Vercel token on merge to \texttt{main}.
        This sidesteps Vercel Hobby's org-private-repo + commit-author restrictions, so all three
        admins' merged work deploys for free.
  \item \textbf{Env vars / secrets} live in Vercel + GitHub Actions, never in the repo.
  \item \textbf{Branch protection:} \texttt{main} requires a PR + 1 review; no direct pushes.
  \item \textbf{Commit identity:} Fatima Abdul Raheem (CEO), \texttt{fatima.abdulraheemdev.17@gmail.com},
        is the committing GitHub account for the \texttt{nurvexthink} org.
  \item \textbf{Commits:} Conventional Commits (\texttt{feat:}, \texttt{fix:}, \texttt{chore:}, \texttt{docs:}).
\end{itemize}

\subsection{Vercel/GitHub deployment notes (researched)}
\begin{itemize}
  \item A Vercel project name can differ from the GitHub username and repo name --- no limitation.
  \item Vercel Hobby cannot deploy private repos owned by a GitHub org, and only the owner's
        commits trigger deploys $\rightarrow$ resolved by public repo + GitHub Actions (chosen),
        or by upgrading to Vercel Pro Team (\textasciitilde\$20/seat/mo) later.
  \item Cloudflare Pages is the fallback host (more generous org/team free tier, rougher Next.js support).
\end{itemize}

\section{Performance \& quality (researched best practices)}
\subsection{Next.js}
\begin{itemize}
  \item Server Components by default; keep \texttt{"use client"} at the leaves.
  \item \texttt{next/image} with \texttt{priority} for the above-the-fold hero (LCP).
  \item Lean on the React Compiler (Next 15+); cache/revalidate \texttt{fetch}.
  \item Target Core Web Vitals (LCP, INP, CLS).
\end{itemize}
\subsection{React Three Fiber / Three.js}
\begin{itemize}
  \item \texttt{frameloop="demand"} --- render only on change.
  \item Mutate via \texttt{useFrame} + refs, never React state in the loop.
  \item Compressed textures (KTX2/Basis), compressed models (GLB + Draco), \texttt{useGLTF.preload()}.
  \item LOD via drei \texttt{<Detailed>}; dispose GPU resources; lazy-load 3D so it never blocks first paint.
  \item Monitor with \texttt{r3f-perf}.
\end{itemize}
\subsection{Supabase / Postgres}
\begin{itemize}
  \item Index columns used in RLS policies and frequent filters.
  \item Test RLS with Studio impersonation before launch.
\end{itemize}
