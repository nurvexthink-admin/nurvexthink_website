\section{Overview}
Replace the per-section static backdrops with a single \textbf{site-wide animated ambient
background}: the \textbf{real NurvexThink mark, extruded into true 3D}, slowly turning like a
physical object on a turntable, floating over slow-moving aurora glows and a faint blueprint
grid. The watermark \emph{is} the hero visual --- the home headline sits directly over it, so
the site has exactly \textbf{one} live WebGL scene.

The effect is \textbf{theme-aware} (opacity/lift per theme; the logo keeps its true colours in
light mode) and reacts subtly to the mouse for depth. It renders \textbf{once}, behind all
public pages, and is suppressed on \texttt{/admin}.

\subsection{Goals}
\begin{itemize}
  \item A premium, branded atmosphere that reinforces the real mark on every public page.
  \item True 3D: depth, beveled edges, metallic shine as the mark turns.
  \item Smooth, GPU-friendly motion that never harms readability or Core Web Vitals.
  \item Fully theme-aware, honoring \texttt{prefers-reduced-motion}.
\end{itemize}

\subsection{Non-goals}
\begin{itemize}
  \item No second live 3D scene (the old hero crystal/canvas was removed in favour of this one).
  \item No per-page configuration or CMS control of the background.
  \item Not shown on admin pages (kept clean and functional).
\end{itemize}

\section{Approach}
The watermark is a real extruded mesh (React Three Fiber), because a flat image cannot show
true perspective, bevels, or specular shine. Everything else (aurora, grid, parallax) stays
\textbf{pure CSS transforms/opacity} on the compositor. Rejected: multiple canvases (hero +
background) --- duplicate GPU cost and two copies of the mark on screen; CSS-3D image tricks ---
no real depth or lighting.

\subsection{Logo fidelity}
\texttt{public/logo.jpeg} is raster on black, so the pipeline is: knock out the background
$\rightarrow$ \texttt{public/logo-mark.png} (transparent, cropped) $\rightarrow$
\textbf{auto-trace with potrace} into two smooth vector path sets (navy N, silver T) stored in
\texttt{src/components/logo-shapes.ts}. The trace was visually verified against the original;
regenerate it if the brand mark ever changes.

\section{Architecture}
\begin{center}
\begin{tabularx}{\textwidth}{l Y}
\toprule
\textbf{Unit} & \textbf{Responsibility}\\
\midrule
\texttt{components/ambient-background.tsx} & The \texttt{fixed inset-0 -z-10
pointer-events-none} layer, \texttt{aria-hidden}. Owns mouse parallax and visibility rules;
returns \texttt{null} on \texttt{/admin*}. Lazy-loads the 3D scene.\\
\texttt{components/ambient-logo-3d.tsx} & The one WebGL scene: extruded logo (SVGLoader
$\rightarrow$ \texttt{ExtrudeGeometry}, bevels), metallic materials, brand-coloured
Lightformer environment, 360\textdegree{} turntable via \texttt{useFrame} + refs.
\texttt{dpr=[1,1.5]}, \texttt{frameloop="demand"} under reduced motion. Disposes geometries on
unmount.\\
\texttt{components/logo-shapes.ts} & The traced vector outlines of the real mark (data only).\\
\texttt{app/globals.css} & Ambient tokens (\texttt{-{}-amb-*}), aurora/grid keyframes,
\texttt{.ambient-*} classes. Size, opacity, per-theme filter, and parallax live on the CSS
wrapper --- the 3D scene stays theme-blind.\\
\texttt{app/layout.tsx} & Mounts \texttt{<AmbientBackground />} once, as the first child of
\texttt{<body>}.\\
\bottomrule
\end{tabularx}
\end{center}

\subsection{Layering (back to front)}
\begin{enumerate}
  \item \textbf{Base} --- existing \texttt{-{}-background} token (painted by \texttt{<body>}).
  \item \textbf{Aurora} --- three large soft radial-gradient blobs, slowly drifting + breathing (CSS).
  \item \textbf{Grid} --- the faint blueprint grid, full-page, edge-masked (CSS).
  \item \textbf{3D logo watermark} --- centred, \texttt{min(72vw, 860px)} wide, slowly turning;
        opacity/lift from theme tokens.
\end{enumerate}

The layer sits at \texttt{-z-10} as a child of \texttt{<body>}: it paints above the body's
background fill but below all in-flow content, so opaque \texttt{bg-card} surfaces naturally
mask it.

\subsection{Mouse parallax}
One rAF-throttled \texttt{pointermove} listener writes normalized
\texttt{-{}-amb-mx}/\texttt{-{}-amb-my} ($-1\ldots1$) onto the layer root; each CSS layer
translates a different magnitude (grid least, aurora most). Not attached when
\texttt{(hover: none)} or \texttt{prefers-reduced-motion: reduce}.

The 3D materials never change per theme --- only the CSS wrapper's opacity/filter do.

% ===================================================================
\section{Performance \& accessibility}
\begin{itemize}
  \item One lazy-loaded WebGL scene site-wide (\texttt{next/dynamic}, \texttt{ssr:false}) ---
        never blocks first paint; dpr capped at 1.5 (the layer is faint; extra resolution is
        wasted GPU).
  \item Turntable animates via \texttt{useFrame} ref mutation --- no React state in the loop.
  \item Aurora/grid animate only \texttt{transform}/\texttt{opacity};
        \texttt{pointer-events: none}; \texttt{aria-hidden}.
  \item Reduced motion: canvas drops to \texttt{frameloop="demand"} (static frame), CSS
        animations freeze via the global guard, parallax listener self-disables.
  \item Manually created geometries are disposed on unmount.
\end{itemize}
